# ZEUS / Olympus Classical Model

## Decisão principal

O modelo canónico do ZEUS passa a ser o dos **12 deuses clássicos**.

Isto significa:

- os deuses formais do sistema são apenas 12;
- os extras continuam a existir como capacidades úteis;
- mas deixam de ser a camada simbólica principal.

## Os 12 clássicos e as suas funções

### Zeus

- orquestração soberana;
- síntese;
- convocação dos outros deuses;
- recomendação final ao Miguel.

### Hera

- governação relacional e institucional;
- compromissos;
- reputação;
- coerência externa;
- enquadramento de responsabilidade.

### Poseidon

- browser;
- rede;
- superfícies externas instáveis;
- interação com sistemas voláteis.

### Deméter

- recursos;
- base documental;
- continuidade material;
- inventário operacional;
- sustentação do trabalho.

### Athena

- estratégia;
- critérios;
- prioridades;
- arquitetura de decisão;
- trade-offs.

### Apollo

- clareza;
- narrativa;
- comunicação;
- inteligibilidade;
- revisão final da expressão pública.

### Artemis

- observação discreta;
- watch;
- recolha seletiva de sinais;
- precisão;
- deteção de padrões.

### Ares

- contraditório;
- risco;
- pressão adversarial;
- cenários de falha;
- robustez sob conflito.

### Aphrodite

- desirability;
- primeira impressão;
- relação humana;
- apelo percebido da experiência.

### Hephaestus

- engenharia;
- build;
- patches;
- implementação;
- automação;
- validação técnica.

### Hermes

- emails;
- drafts;
- mensagens;
- triagem;
- handoff;
- transporte operacional de informação.

### Hestia

- memória operacional;
- estado interno;
- continuidade;
- contexto persistente;
- centro estável do sistema.

## Porque Hestia e não Dionysus

Para o ZEUS, Hestia faz mais sentido porque o sistema precisa primeiro de:

- centro;
- estabilidade;
- contexto;
- memória persistente;
- continuidade operacional.

Dionysus só faria mais sentido se a exploração criativa caótica fosse uma função central formal, e esse não é o caso neste momento.

## Absorção dos extras

Os extras deixam de ser o panteão principal e passam a ser absorvidos assim:

- Themis -> Hera + Athena
- Mnemosyne -> Hestia
- Harmonia -> Apollo
- Daedalus -> Athena + Hephaestus
- Argus -> Artemis
- Plutus -> Deméter + Athena
- Prometheus -> Athena + Hephaestus

## Regra operacional

Os extras podem continuar a existir como:

- módulos;
- capacidades;
- helpers;
- pareceres especializados.

Mas o Olympus formal do ZEUS deve mostrar apenas:

1. Zeus
2. Hera
3. Poseidon
4. Deméter
5. Athena
6. Apollo
7. Artemis
8. Ares
9. Aphrodite
10. Hephaestus
11. Hermes
12. Hestia

## Implicação prática para o repo

O caminho recomendado não é apagar já tudo o que existe.

O caminho certo é:

1. manter os módulos úteis já implementados;
2. reclassificá-los como subfunções internas;
3. apresentar no advisor, docs e routing apenas os 12 clássicos como camada simbólica principal.
