# Governance Layer — Founder Command Center

## Objetivo

Garantir que o AtelierOS e o Founder Command Center evoluem de forma:
- segura;
- consistente;
- auditável;
- sustentável;
- alinhada com validação humana.

---

# Princípio Central

A IA pode:
- analisar;
- propor;
- preparar;
- organizar;
- automatizar parcialmente.

A IA NÃO pode executar ações críticas sem aprovação humana.

---

# Approval Gates

## Necessitam aprovação humana obrigatória

- deploys;
- alterações críticas de memória;
- envio de emails críticos;
- alterações contratuais;
- submissões oficiais;
- eliminação de ficheiros;
- alterações financeiras;
- alterações irreversíveis.

---

# Memory Governance

## Regras

- entidades canónicas obrigatórias;
- evitar duplicação;
- evitar corrupção semântica;
- evitar loops;
- preservar histórico;
- manter audit trail.

---

# Agent Governance

## Nenhum agente pode:

- agir fora do seu scope;
- alterar runtime crítico;
- ignorar critic agents;
- modificar governance;
- executar ações destrutivas.

---

# Security Principles

## O sistema deve:

- usar secrets via .env;
- nunca guardar API keys no código;
- manter logs;
- manter histórico;
- permitir rollback;
- isolar tools sensíveis;
- limitar permissões.

---

# Human-in-the-loop

O sistema é:
- assistido por IA;
- supervisionado por humano.

Objetivo:
- aumentar capacidade;
- reduzir carga operacional;
- reduzir caos;
- acelerar execução;
- manter controlo humano.

---

# Runtime Philosophy

O Founder Command Center é:
- runtime contínuo;
- ambiente operacional vivo;
- sistema cognitivo governado.

Não é:
- chatbot;
- dashboard passivo;
- automação sem supervisão.

---

# Regra Final

Governance > velocidade.

Sistemas sem governance degradam.
